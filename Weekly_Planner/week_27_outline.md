# Week 27: Final Project — Working Session 1

## Course
**Python for Data Science (Python DS)** — Learn and Help

## Topic
Launching the Final Project. Students explore data science certifications, hackathons, and project ideas to frame their capstone work. The centerpiece is the **Pathways to Success** project — a collaborative, data-driven study of student opportunities across all 50 U.S. states.

## Prior Knowledge
Students bring the full skill set built across Weeks 1–26:
- Python fundamentals, OOP, and data encapsulation (Weeks 1–5)
- Data types, sources, Excel analysis (Weeks 6–8)
- NumPy and pandas — data loading, cleaning, EDA, aggregation, time series (Weeks 9–20)
- Matplotlib, Seaborn, and Plotly visualizations (Weeks 21–24)
- HTML structure and the DOM (Week 25)
- Web scraping with Beautiful Soup (Week 26)

## Learning Objectives
By the end of this session, students will be able to:
1. Identify free certifications (Kaggle, freeCodeCamp, Google) that validate their Python DS skills
2. Describe what a data science hackathon is and name at least two beginner-friendly competitions
3. Apply the **Problem → Data → Insight** framework to generate a personal project idea
4. Locate and evaluate a real-world dataset using Kaggle, Data.gov, or similar sources
5. Explain the five-phase structure of the Pathways to Success final project
6. Identify their assigned U.S. states and locate those rows in the starter Excel dataset
7. Review existing data for quality — spot missing fields, broken links, or inaccurate entries
8. Plan what new programs they will add for their states in the coming weeks

## Key Vocabulary

| Term | Definition |
|---|---|
| **Capstone Project** | A culminating project that demonstrates all skills learned throughout a course |
| **Hackathon** | A timed competition where participants solve a problem using data or code |
| **Certification** | A credential awarded after completing a course or passing an assessment |
| **EDA (Exploratory Data Analysis)** | The process of summarizing and visualizing a dataset to understand its structure |
| **Data Enrichment** | Adding missing or new records to improve the completeness of an existing dataset |
| **Data Quality** | The accuracy, completeness, and consistency of data in a dataset |
| **Research Question** | A specific, answerable question that guides a data science project |
| **Starter Dataset** | A pre-collected dataset provided as a starting point for further review and analysis |
| **Stakeholder** | A person or group who has an interest in the findings of a project (e.g., educators, parents, students) |
| **Leaderboard** | A ranked scoreboard used in competitions to compare participant results |
