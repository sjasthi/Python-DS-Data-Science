What is HTML?

HTML = HyperText Markup Language

HTML is the skeleton of every web page. It tells the browser what content is on the page and how it is organized. As data scientists, we read HTML to extract data from websites.

Tags — The Building Blocks

HTML uses tags — labels wrapped in angle brackets < >. Most tags come in pairs:

<tagname>Content goes here</tagname>

The closing tag has a forward slash: </tagname>

Common Tags You'll See

TagWhat It Does
<h1> to <h6>Headings (h1 = largest, h6 = smallest)
<p>Paragraph of text
<a href="...">Link to another page
<img src="...">Image (self-closing — no closing tag needed)
<ul> + <li>Unordered (bullet) list
<ol> + <li>Ordered (numbered) list
<br>Line break (self-closing)
<hr>Horizontal line (self-closing)

The HTML Document Structure

Every HTML page follows this skeleton:

<!DOCTYPE html>
<html>
  <head>
    <title>Page Title</title>
  </head>
  <body>
    <h1>Hello World!</h1>
    <p>Content goes here.</p>
  </body>
</html>

The HTML Tree (DOM)

HTML is organized like a family tree. Tags can be parents, children, and siblings:

html
├── head
│   └── title
└── body
    ├── h1
    ├── p
    └── ul
        ├── li
        ├── li
        └── li

This tree is called the DOM (Document Object Model) — it's what Beautiful Soup navigates to find data!

Attributes — Extra Info on Tags

Attributes go inside the opening tag and provide additional details:

<a href="https://google.com">Click here</a>
<img src="cat.jpg" alt="A cute cat">
<div id="header" class="main">...</div>
AttributePurpose
idA unique name for one element on the page
classA group label — many elements can share the same class
hrefWhere a link goes
srcSource file for images or media
🔗 Beautiful Soup Connection

id and class are the two most important attributes for web scraping. Next week you'll write code like:
soup.find('div', id='main-content')
soup.find_all('p', class_='article')

Nesting — Tags Inside Tags

Tags are often nested inside each other, like boxes within boxes:

<div class="card">
  <h2>Product Name</h2>
  <p>Price: <span class="price">$29.99</span></p>
  <a href="/buy">Buy Now</a>
</div>

Rule: Tags must close in the reverse order they opened — like unstacking plates.

Containers: <div> and <span>

<div> — a block container (takes full width, stacks vertically)

<span> — an inline container (wraps just a few words within a line)

Websites use <div> and <span> with class and id attributes to organize content — these are your primary targets when scraping!

HTML Tables — A Data Scientist's Best Friend

A huge amount of data on the web lives inside HTML tables. Understanding the table tags is critical for scraping.

Table Tag Anatomy

TagStands ForPurpose
<table>TableWraps the entire table
<tr>Table RowOne horizontal row
<th>Table HeaderHeader cell (bold, centered)
<td>Table DataRegular data cell
<thead>Table HeadGroups header rows (optional)
<tbody>Table BodyGroups data rows (optional)

How a Table Maps to a DataFrame

This HTML table:

<table>
  <tr>
    <th>City</th>
    <th>Population</th>
    <th>State</th>
  </tr>
  <tr>
    <td>Atlanta</td>
    <td>498,715</td>
    <td>GA</td>
  </tr>
  <tr>
    <td>Boston</td>
    <td>675,647</td>
    <td>MA</td>
  </tr>
</table>

…becomes this Pandas DataFrame:

     City  Population State
0  Atlanta    498,715    GA
1   Boston    675,647    MA
🐍 Beautiful Soup Preview

Next week you'll extract tables with code like this:

from bs4 import BeautifulSoup

soup = BeautifulSoup(html, 'html.parser')
table = soup.find('table')
rows = table.find_all('tr')

for row in rows:
    cells = row.find_all(['th', 'td'])
    data = [cell.text for cell in cells]
    print(data)
