## 1. How to Come Up With a Project Idea

Coming up with a project idea can feel scary — but it's actually a creative process! Think of it like choosing a science fair topic. Follow these four steps:

---

### 🔎 Step 1 — Ask "What Do I Care About?"

Great data science projects start with **curiosity**. Ask yourself:

- What problems do I see in my school, neighborhood, or community?
- What hobby, sport, or topic could I explore with data?
- What would I Google on a rainy afternoon?

> **Example**: *"I love basketball — can I predict which teams will make the playoffs?"*  
> **Example**: *"My city has bad air quality — can I visualize which months are worst?"*

---

### 🧩 Step 2 — Use the "Problem → Data → Insight" Framework

Every data science project follows this pattern:

```
Problem (What question am I asking?)
    ↓
Data (Where can I find numbers that answer it?)
    ↓
Analysis (What patterns or trends do I see?)
    ↓
Insight (What should someone do differently because of my findings?)
```

| Problem | Data Source | Possible Insight |
|---|---|---|
| Are video games more violent than before? | ESRB ratings dataset | Genre trends over decades |
| Which US states have the most tornadoes? | NOAA weather data | Seasonal tornado patterns |
| Is fast food getting more expensive? | Menu price datasets | Price inflation by chain |
| Which Spotify genres are most popular at school? | Spotify Charts API | Local vs. global trends |
| Does temperature affect school attendance? | Weather + school CSV | Best months to be absent! |

---

### 🛠️ Step 3 — Match Your Idea to Your Skills

You've learned a LOT this year! Make sure your project uses at least **3 skills** from class:

| Skill | What You Learned | Can Use For |
|---|---|---|
| pandas | Load, clean, filter data | Any CSV or Excel dataset |
| matplotlib / seaborn | Charts and graphs | Visualizing trends |
| numpy | Math on data | Averages, statistics |
| Beautiful Soup | Scraping websites | Live / updated data |
| EDA | Exploring a dataset | Finding surprises in data |
| HTML | Web pages | Presenting your project |

---

### ✅ Step 4 — Pass the "Can I Do It?" Test

Before committing to your idea, answer YES to all three:

- [ ] Can I find a **real dataset** for this topic? (See Section 4)
- [ ] Can I write a **clear question** my project will answer?
- [ ] Can I show my results as **at least 2 charts** or visualizations?

If you answered YES to all three — **you have a project!** 🎉

Use this timeline for Weeks 27–30:

```

---

### 📝 Project Requirements Checklist

Your final project should include:

- [ ] A **clear research question** (1–2 sentences)
- [ ] A **real dataset** with at least 100 rows
- [ ] **Data loading** with pandas (`pd.read_csv()` or similar)
- [ ] **Data cleaning** — handle at least one messy data issue
- [ ] **EDA** — at least 5 descriptive statistics (`.describe()`, `.value_counts()`, etc.)
- [ ] **At least 3 visualizations** (bar chart, line chart, or scatter plot)
- [ ] A **written summary** — 3–5 sentences explaining your findings
- [ ] All code in a **Google Colab notebook** (shared with instructor)

---

---

### 💡 Pro Tips for Finding Data

1. **Search Kaggle first** — it has the most beginner-friendly datasets with built-in notebooks
2. **Look for CSV files** — they're easiest to load with `pd.read_csv()`
3. **Check the number of rows** — aim for 100–10,000 rows (not too small, not overwhelming)
4. **Read the data description** — understand what each column means before you start
5. **Download and upload to Colab** — or use the Kaggle API directly in notebooks

## 7. Showcasing Your Work

Great data science work deserves a great audience! Here's how to share your project with the world:

---

### 📓 Option 1 — Google Colab Notebook (Required)

Your project must be in a **Colab notebook**. Make sure it's well-organized:

```
📓 My_Project_Name.ipynb
├── Section 1: Introduction & Research Question
├── Section 2: Dataset Description
├── Section 3: Data Loading & Cleaning
├── Section 4: Exploratory Data Analysis (EDA)

└── Section 6: Summary & Conclusions

Share with your instructor at: Google Classroom submission

---

### 🌐 Option 2 — Kaggle Notebook (Bonus)

Publish your notebook on Kaggle — it becomes part of your **public data science portfolio**!

1. Go to [kaggle.com](https://www.kaggle.com) → Sign In
2. Click **"Create" → "Notebook"**
3. Paste your Colab code
4. Click **"Share"** → Make Public
5. Copy the link and add it to your submission

---

### 💻 Option 3 — GitHub Repository (Advanced Bonus)

Post your project on GitHub like a real developer:

1. Create a free account at [github.com](https://github.com)
2. Click **"New Repository"** — name it `my-data-science-project`
3. Upload your `.ipynb` notebook and dataset
4. Write a **README.md** that explains your project
5. Share your GitHub link!

---

### 🎤 Option 4 — Class Presentation (Week 30 — Required!)

During **Week 30's Final Demonstration**, you'll present your project to the class. Prepare:

- [ ] **Slide deck** (5–8 slides) OR open your notebook directly

- [ ] **Your "So What?"** — why does your finding matter?

### 🐍 Code Snippets You'll Need

**Loading a CSV from Google Drive in Colab:**
```python
import pandas as pd

# Upload CSV to Colab first, then:
df = pd.read_csv('your_file.csv')
print(df.shape)
df.head()
```

**Quick EDA:**
```python
df.describe()          # Summary statistics
df.info()              # Column types + nulls
df.isnull().sum()      # Count missing values
df['column'].value_counts()  # Frequency count
```

**Basic Chart (matplotlib):**
```python
import matplotlib.pyplot as plt

plt.figure(figsize=(10, 6))
plt.bar(df['category'], df['value'], color='steelblue')
plt.title('My Chart Title', fontsize=16)
plt.xlabel('Category')
plt.ylabel('Value')
plt.xticks(rotation=45)
plt.tight_layout()
plt.show()
```

**Basic Chart (seaborn):**
```python
import seaborn as sns

sns.histplot(df['column'], kde=True, color='coral')
plt.title('Distribution of Column')
plt.show()
```

**Beautiful Soup — Scraping a Table:**
```python
import requests
from bs4 import BeautifulSoup
import pandas as pd

url = 'https://example.com/data-table'
response = requests.get(url)
soup = BeautifulSoup(response.text, 'html.parser')
table = soup.find('table')
df = pd.read_html(str(table))[0]
print(df.head())
```
